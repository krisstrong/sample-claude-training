8.1 Why Documentation Matters (And Why It's Hard)

IT documentation is perpetually underdone. Not because staff don't understand its value — they do — but because writing it manually takes time that engineers don't have in the middle of deployments, incidents, and support queues.

⌛

The Old Way

Write everything from scratch, after hours, when the system is already in production and you're already exhausted

🤖

With Claude

Draft in minutes. Your job shifts from writing to reviewing, refining, and approving

✅

The Result

More documentation, better documentation, updated regularly because the barrier is low

Mindset Shift
Stop thinking of Claude as a writing tool you use when you have spare time. Think of it as a documentation partner you involve while the work is happening — draft the README as you write the script, draft the runbook as you build the procedure.

8.2 README Files and Technical Guides

A README is the entry point for any script, module, or tool. Claude generates READMEs from code or from a brief verbal description — whichever you have available.

Generating a README from code

Claude — Technology Services Workspace
You
Write a README.md for this PowerShell module — it manages Active Directory user provisioning for new staff and students at Acadia University. Here's the code: [paste of Provision-ADUser.psm1]
Claude
Here's a README for the AD User Provisioning module:

AD User Provisioning Module

Automates Active Directory user account creation, modification, and deprovisioning for Acadia University Technology Services. Supports bulk imports and integrates with the university's HR data feeds.

Requirements

  • Windows PowerShell 5.1 or PowerShell 7+
  • ActiveDirectory module (RSAT: Active Directory Domain Services)
  • Domain admin credentials or delegated OU permissions
  • Write access to \\acadiau.ca\NETLOGON\scripts

Installation

Import-Module .\Provision-ADUser.psm1

Usage

Provision a single user

New-AcadiaUser -FirstName "Jane" -LastName "Smith" -Role "Faculty" -Department "Computer Science"

Deprovision a user

Remove-AcadiaUser -Username "jsmith" -ArchiveHome

Parameters

See Get-Help New-AcadiaUser -Full for complete parameter documentation.

Logging

All provisioning actions are logged to C:\Logs\AD-Provisioning\ with timestamps and the performing account. Log rotation occurs at 30 days.

Shall I add a Troubleshooting section or a Parameters table?

Updating a README when code changes

READMEs go stale when scripts gain new features. Claude makes updates easy — paste the current README, describe the change, and ask for the update.

Bulk Import from CSV

Use the --BulkImport flag to provision multiple users from a CSV file in a single operation.

New-AcadiaUser --BulkImport users.csv [-WhatIf] [-SendWelcomeEmail]

CSV Format

The import CSV must include the following columns (headers are case-insensitive):

FirstName,LastName,Role,Department,Email
Jane,Smith,Faculty,Computer Science,jsmith@acadiau.ca
Bob,Jones,Staff,Technology Services,bjones@acadiau.ca

8.3 Runbooks and SOPs

Runbooks and standard operating procedures are critical for incident response — especially when the person who built a system isn't available. Claude produces clear, structured procedures from a verbal description or from informal notes.

Example: Email relay service restart runbook

Prompt to Claude: "Write a runbook for restarting the email relay service on our Windows servers. Include pre-checks, step-by-step restart procedure, verification steps, and a rollback procedure."

RUNBOOK

Email Relay Service Restart — smtp-relay-01 / smtp-relay-02

Technology Services — Last updated: 2026-06-28 — Owner: Infrastructure Team

Pre-Conditions

  • Confirm the restart is approved — check the change management calendar for active freeze windows
  • Notify #ts-alerts Slack channel: "Restarting smtp-relay-01 at [time] — brief email disruption expected"
  • Verify the backup relay (smtp-relay-02) is healthy: Test-NetConnection smtp-relay-02 -Port 25
  • Confirm you have admin credentials for the target server

Restart Procedure

  1. Connect to the server via RDP or PowerShell remoting.

    Enter-PSSession -ComputerName smtp-relay-01 -Credential (Get-Credential)

  2. Check current queue depth before stopping the service. If the queue exceeds 500 messages, wait for it to drain or escalate before proceeding.

    Get-Message -Server smtp-relay-01 | Measure-Object | Select-Object Count

  3. Stop the SMTP relay service gracefully.

    Stop-Service -Name "MSExchangeTransport" -Force
# Wait for the service to fully stop before proceeding
(Get-Service "MSExchangeTransport").WaitForStatus('Stopped', '00:01:00')

  4. Start the service.

    Start-Service -Name "MSExchangeTransport"
(Get-Service "MSExchangeTransport").WaitForStatus('Running', '00:01:00')

  5. Verify the service is accepting connections on port 25.

    Test-NetConnection smtp-relay-01 -Port 25

    Expected result: TcpTestSucceeded : True. If False, proceed to Rollback.

  6. Send a test message to confirm end-to-end delivery. Use the internal mailflow test account and verify receipt within 2 minutes.

Verification

  • Service status shows Running in Services panel
  • Port 25 accepts connections from both internal and relay-authorized IPs
  • Test email delivered successfully
  • Event log shows no errors in the last 5 minutes: Get-EventLog -LogName Application -Source *Transport* -Newest 20
  • Post resolution message to #ts-alerts: "smtp-relay-01 restart complete and verified"

Rollback

If the service does not start or port 25 is not responding after step 5:

  1. Redirect all mail flow to smtp-relay-02 via DNS TTL reduction or load balancer configuration
  2. Review the Application event log on smtp-relay-01 for service start errors
  3. Escalate to Infrastructure Team lead if not resolved within 15 minutes
  4. Open a P1 incident ticket in the helpdesk with the event log excerpt attached

Converting informal notes to a formal SOP

Claude
You
Convert this informal Confluence note into a formal SOP for Acadia Technology Services. The audience is any IT staff member, not just the person who wrote the original note.

"to provision a new student, go into AD, find the students OU, copy an existing student account. change the name, username, and set the temp password to Welcome1! then email them. also set the office 365 license via the admin portal – need an E1 license. sometimes you need to wait 10 minutes for it to sync."
Claude

Here's the formalized SOP. I've also flagged one concern: Welcome1! as a temp password is predictable — I've replaced it with a recommendation to use a random password generator. You may want to review that with your security policy.

SOP: New Student Account Provisioning
SOP-IT-007   Identity & Access Management

Purpose: Ensure consistent, secure provisioning of Active Directory and Microsoft 365 accounts for new students.

Scope: All Technology Services staff performing student onboarding.

Prerequisites: Active Directory admin access; Microsoft 365 Admin Center access with license assignment permissions.

Procedure:

  1. Open Active Directory Users and Computers and navigate to the Students OU (acadiau.ca > Students > [Year])
  2. Right-click an existing student account → Copy. This copies group memberships and OU placement.
  3. Enter the new student's first name, last name, and username (format: flastname)
  4. Set a temporary password using the department password generator. Do not use a predictable default. The password must meet complexity requirements.
  5. Check User must change password at next logon
  6. Email the student their username and temporary password using the provisioning email template in the shared mailbox
  7. Assign Microsoft 365 E1 license: open Microsoft 365 Admin Center → Users → Active Users → [student] → Licenses and apps
  8. Wait 10–15 minutes for Entra ID synchronization before confirming with the student that the account is active

Verification: Confirm the student can sign in to office.com with their credentials.

8.5 User-Facing Help Articles

Technology Services publishes help articles for students, faculty, and staff. Claude writes clearly for non-technical audiences — you just need to specify who the reader is.

The audience specification matters enormously

Prompt Technique
Always state the audience explicitly. "Write for a first-year student with no technical background" produces a fundamentally different document than "Write for a faculty member comfortable with basic technology." Claude adjusts vocabulary, assumed knowledge, and step granularity accordingly.

For Students

Use phrases like: "Write for a first-year student who has never connected to a university network before. Avoid jargon. Define any technical term you use."

Claude will: use short sentences, include screenshots callouts like "(you'll see a blue button labeled 'Connect')", avoid acronyms, and assume zero prior knowledge.

For Faculty

Use phrases like: "Write for a faculty member comfortable with basic technology but not IT administration. Assume they know what email and VPN mean."

Claude will: use intermediate vocabulary, skip basic definitions, focus on the why alongside the how.

Example: VPN help article for students

Prompt to Claude: "Write a help article for Acadia students explaining how to connect to the university VPN on Windows 11. Use simple language. The VPN client is Cisco Secure Client."

Connecting to the Acadia VPN on Windows 11

The Acadia VPN (Virtual Private Network) lets you access university resources — like library databases and internal systems — when you're off campus. Think of it as a secure tunnel between your computer and the Acadia network.

Before You Start

  • You need the Cisco Secure Client installed. If you don't have it yet, download it from it.acadiau.ca/vpn and run the installer.
  • You'll need your Acadia username and password (the same ones you use for email and Moodle).
  • Make sure you have internet access before connecting — the VPN goes through your regular internet connection.

How to Connect

  1. Click the Start button (the Windows logo in the taskbar) and search for Cisco Secure Client. Click it to open the app.
  2. In the text box, type vpn.acadiau.ca and click Connect.
  3. A login window will appear. Enter your Acadia username (e.g., jsmith) and your password.
  4. If you have two-factor authentication set up, you'll receive a push notification on your phone. Approve it.
  5. Once connected, the Cisco icon in your taskbar will show a small padlock symbol. You're now on the Acadia network.

How to Disconnect

Right-click the Cisco Secure Client icon in the taskbar (near the clock) and select Disconnect. Always disconnect when you're done — staying connected when you don't need it can slow down your internet.

Having Trouble?

  • The connection keeps dropping: Try a different network (e.g., switch from Wi-Fi to a wired connection).
  • Wrong username or password: Try logging into webmail.acadiau.ca first to confirm your credentials work.
  • Still not working: Contact the Technology Services Help Desk at helpdesk@acadiau.ca or visit us in the BAC Building, Room 107.

8.6 Applying the Acadia Theme to Deliverables

All Technology Services documents shared externally or published on internal portals should use the Acadia house visual theme. Claude has skills built specifically for this.

The two skills you use

/theme

Applies the Acadia visual identity to any deliverable: Acadia Blue (#004077), Acadia Red (#C41424), Montserrat headings, Arial body text. Works for HTML pages, Word documents, PDF reports, and slides.

Use whenever asked to "make it on-brand," "apply our theme," or "use our colors."

/file-naming

Names the output file in the Technology Services standard convention:

Technology Services - [Short Name] - YYYYMMDD.[ext]

Consistent file naming makes documents findable in shared drives and archives.

Branded HTML report workflow

  1. Ask Claude to draft the content. Focus the prompt on what the document should contain, not how it should look.

    "Write a monthly IT security summary report for June 2026. Include: open vulnerabilities, patches applied, incidents handled, and recommendations."

  2. Invoke /theme. Claude transforms the plain content into a fully styled HTML document using the Acadia color palette, typography, and component library — callouts, tables, cards, and all.

  3. Invoke /file-naming. Claude names the output file following the Technology Services convention: Technology Services - IT Security Summary June 2026 - 20260628.html

  4. Review the file in your browser. Open it in Chrome or Edge to confirm layout, check for any data errors, and make final adjustments before distributing.

8.7 Documentation Workflow Tips

These techniques make Claude most effective as a documentation partner across the full lifecycle of a document — not just first drafts.

Iterating on a draft

Claude holds your document in context for the entire session. Iterate conversationally:

Claude
You
Make section 2 more concise — it's too wordy for a runbook. Cut it to the key action items only.
Claude
Revised section 2 — reduced from 180 words to 62, keeping only the actionable steps. [revised section shown]
You
Add a troubleshooting FAQ at the end. Include the three most common failure modes for this service.
Claude
Added a Troubleshooting FAQ section covering: (1) service won't start after restart, (2) port 25 blocked by firewall, (3) queue stuck in retry state. [FAQ section shown]

Improving existing documentation

Paste any existing document and ask Claude to audit or improve it:

  • "This runbook is unclear in places. Rewrite it for clarity and consistency."
  • "What's missing from this disaster recovery procedure? Are there any steps a reader might get stuck on?"
  • "This SOP uses inconsistent terminology — sometimes 'user account', sometimes 'login', sometimes 'identity'. Standardize it to 'user account' throughout."
  • "Rate this document for clarity on a scale of 1–10 and list the three biggest improvements to make."

Format conversion

Claude converts documentation between formats without losing structure:

From To Prompt
Confluence wiki markup Markdown "Convert this Confluence wiki markup to clean Markdown."
Markdown HTML (themed) "Convert this Markdown README to an HTML page using the Acadia theme. Invoke /theme."
Bullet point notes Formal SOP "Expand these bullet points into a full SOP with numbered steps and pre/post conditions."
Technical runbook User guide "Rewrite this technical runbook as a student-facing self-help guide. Remove all IT-specific commands."
Email thread Decision log "Extract the key decisions and rationale from this email thread and format as a decision log."

Documentation quality checklist

Before publishing any Claude-generated document, review it against this checklist:

Pre-Publication Review

  • All technical steps have been tested and verified in the actual environment
  • Commands, paths, and server names are correct for the Acadia environment (not generic placeholders)
  • The document has a clear owner and last-updated date
  • Links to internal systems or related documents are accurate
  • Sensitive information (passwords, internal IPs, credentials) has been removed
  • The audience matches the document's language and assumed knowledge level
  • The file is named using the Technology Services naming convention
  • A second pair of eyes has reviewed the procedure, especially for incident response content
Always Verify Technical Steps
Claude generates documentation based on its training data and what you provide. It does not have access to your live systems. Always test every command and procedure in a non-production environment before publishing a runbook or SOP. Claude is excellent at producing the structure and language; you are responsible for the accuracy of the technical content.
