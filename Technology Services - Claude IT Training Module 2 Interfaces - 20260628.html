Getting Started — Desktop App, Terminal & VS Code

This module gets you operational on all three Claude interfaces. By the end you will have the Desktop App installed, the CLI running from your terminal, and the VS Code extension active in your editor — and you will know which tool to reach for on any given task.

2.1 The Desktop App

The Desktop App is Claude's flagship interface — a full GUI application for Windows and macOS that supports rich conversations, file attachments, Projects, and model selection. It is the best starting point for anyone new to Claude, and it remains the tool of choice for long research tasks, multi-document analysis, and team collaboration through Projects.

Installation

  1. Download the installer. Navigate to claude.ai/download in your browser. Anthropic provides installers for both Windows (.exe) and macOS (.dmg). Select the appropriate version for your machine.

  2. Run the installer. On Windows, launch the downloaded .exe file. You may see a UAC prompt — click "Yes" to allow installation. On macOS, drag Claude to your Applications folder from the mounted DMG.

  3. Sign in to your Anthropic account. When the app opens, you will be prompted to sign in. Use your Anthropic account credentials. If your department has a Teams plan, sign in with the account invited to that workspace.

  4. Select your starting model. On first launch, the app defaults to Claude Sonnet. This is appropriate for most Technology Services work. You can change the model in any conversation using the model picker in the top toolbar.

  5. Start a new conversation. Click the "New Chat" button in the top-left sidebar. Type your first message in the input box at the bottom of the screen and press Enter or click the send button.

Key Desktop App Features

Projects

A Project is a persistent workspace that maintains a shared system prompt and file context across multiple conversations. Create a "Technology Services IT Ops" Project and give it a system prompt like "You are assisting an IT team at a Canadian university. Our environment includes Windows Server 2022, Active Directory, Cisco networking, and Microsoft 365."

Every new chat you open inside that Project inherits this context automatically — you never have to re-explain your environment.

File Attachments

Drag and drop any file into the chat window, or click the paperclip (📎) icon to browse. Claude can read and reason about:

  • Text files, logs, and CSVs
  • Word documents and PDFs
  • Code and configuration files
  • Images (screenshots of errors, diagrams)

Model Selection

The model picker dropdown in the top bar lets you switch between Haiku, Sonnet, and Opus for any conversation. A rule of thumb for Technology Services use:

  • Haiku — simple lookups, format conversion
  • Sonnet — most daily tasks (default)
  • Opus — complex architecture or deep analysis

Conversation Management

All conversations are saved and searchable in the left sidebar. You can rename conversations for easy retrieval, star important ones, and scroll back through the full history of any session.

Note that each conversation has its own context window — starting a new conversation always resets what Claude "knows." Use Projects to share persistent context.

Project System Prompt Tip
When you create a new Project, set its system prompt to describe your IT environment. Include: operating systems in use, key software (AD, M365, Cisco IOS version), any standards you follow (ITIL, CIS Benchmarks), and your preferred output style (e.g., "Always include error handling in PowerShell scripts"). Claude will apply this context to every conversation in the Project.

2.2 The Terminal / CLI

The Claude CLI (often called Claude Code in documentation) is a command-line tool that brings Claude directly into your shell workflows. For IT professionals already living in PowerShell or Bash, this is often the fastest way to get a quick answer, analyse piped output, or integrate Claude into a script pipeline.

Installation

Prerequisite
The Claude CLI requires Node.js version 18 or later. Run node --version to check. If Node.js is not installed, download it from nodejs.org — the LTS version is recommended. You will also need an Anthropic API key, available from your account at console.anthropic.com.

  1. Verify Node.js is installed. Open PowerShell or your terminal and run:

    PowerShellVerification
    node --version
npm --version

    Both commands should return version numbers. If either returns an error, install Node.js first.

  2. Install the Claude CLI globally.

    PowerShell / npmInstall
    npm install -g @anthropic-ai/claude-code

    The -g flag installs the claude command globally so it is available in any terminal session.

  3. Set your API key as an environment variable. The CLI authenticates via your Anthropic API key. Set it for your current session, or add it to your profile permanently:

    PowerShell — Current session onlyAuth
    $env:ANTHROPIC_API_KEY = "sk-ant-your-key-here"
    PowerShell — Add permanently to profileAuth (persistent)
    # Open your PowerShell profile in Notepad, then add the line below
notepad $PROFILE

# Add this line to the profile file:
$env:ANTHROPIC_API_KEY = "sk-ant-your-key-here"

  4. Verify the installation. Run the following to confirm the CLI is working:

    PowerShellVerify
    claude --version

    You should see the installed version number printed to the console.

  5. Run your first query.

    PowerShellFirst run
    claude "What PowerShell command lists all installed software on this machine?"

Basic Usage Patterns

Command Pattern What it does
claude Opens an interactive REPL session. You can have a back-and-forth conversation, maintaining context between turns, until you type exit.
claude "your question here" Single-shot query. Claude responds once and exits. Ideal for quick lookups in the middle of other work.
cat file.txt | claude "summarise this" Pipes file content into Claude as context. Replace cat with any command that produces text output.
Get-Content error.log | claude "identify the root cause" PowerShell equivalent of the above — pipe any cmdlet output directly to Claude.
claude --model claude-haiku-4-5 "quick question" Force a specific model for this query. Useful when you want Haiku's speed for simple tasks or Opus for complex analysis.

CLI Session Example

PowerShell Terminal Session Example 
PS C:\Scripts> claude "What PowerShell command lists all installed software on this machine?"

Claude:
There are two reliable approaches depending on what you need:

Option 1 — Registry query (fast, includes most software):
Get-ItemProperty HKLM:\Software\Microsoft\Windows\CurrentVersion\Uninstall\* |
    Select-Object DisplayName, DisplayVersion, Publisher, InstallDate |
    Where-Object { $_.DisplayName } |
    Sort-Object DisplayName

Option 2 — WMI query (slower but more complete):
Get-CimInstance -ClassName Win32_Product |
    Select-Object Name, Version, Vendor, InstallDate |
    Sort-Object Name

Note: Win32_Product triggers a Windows Installer consistency check on each
query which can be slow and generates Application log entries. The registry
query is preferred for inventory scripts run frequently.

PS C:\Scripts> Get-ItemProperty HKLM:\Software\Microsoft\Windows\CurrentVersion\Uninstall\* |
    Select-Object DisplayName, DisplayVersion | Where-Object { $_.DisplayName } |
    Export-Csv ".\installed-software.csv" -NoTypeInformation

PS C:\Scripts> # CSV exported successfully

Key CLI Flags

Flag Purpose Example
--model Specify which Claude model to use for this query claude --model claude-opus-4-8 "analyse this architecture"
--output-format Control output format: text (default), json, or stream-json for piping to other tools claude --output-format json "list 5 AD audit checks"
--system Provide a system prompt for this session, shaping Claude's behaviour and persona claude --system "You are a Windows Server expert. Be concise."
--version Print the installed version of the Claude CLI claude --version
--help List all available flags and usage information claude --help
CLI Naming Note
In Anthropic's documentation you may see the tool referred to as Claude Code, while the actual installed command is simply claude. They are the same tool. When documentation says "run Claude Code," it means run claude from your terminal.

2.3 The VS Code Extension

If your IT work involves writing scripts, maintaining configuration management code, developing automation, or doing code-heavy documentation, the VS Code extension is your most powerful option. Claude can see your entire workspace, understand the code you have open, and offer inline suggestions without you ever leaving the editor.

Installation

  1. Open the Extensions sidebar. In VS Code, press Ctrl+Shift+X (Windows/Linux) or Cmd+Shift+X (macOS) to open the Extensions view, or click the Extensions icon in the Activity Bar on the left.

  2. Search for Claude. In the search box at the top of the Extensions sidebar, type Claude. Look for "Claude for VS Code" published by Anthropic. Verify the publisher name to avoid installing unofficial alternatives.

  3. Click Install. Click the blue Install button next to the official extension. VS Code will download and activate it automatically — no restart required in most cases.

  4. Sign in or enter your API key. After installation, click the Claude icon that appears in the Activity Bar (left sidebar). You will be prompted to authenticate with your Anthropic account or paste an API key. Follow the prompts.

  5. Open a project folder. For best results, open the root folder of your project or scripts repository (Ctrl+K, Ctrl+O) rather than individual files. Claude reads the workspace structure and can reference any file you have open.

The VS Code extension adds a Claude panel to the Activity Bar. Claude can see the currently open file and your full workspace — here it has identified three error handling issues in a backup script and cited specific line numbers.

Key Extension Features

Sidebar Chat Panel

Open with Ctrl+Shift+A or click the Claude icon (🤖) in the Activity Bar. Ask questions about your open file, the entire workspace, or general IT topics — Claude's responses appear inline in the panel with full conversation history for the session.

Example prompts that work well:

  • "Explain what this function does"
  • "Add parameter validation to the current file"
  • "Find any hardcoded credentials or secrets in this workspace"

Inline Code Assistance

Highlight any block of code in the editor, then press Ctrl+Shift+C to open an inline prompt. Type your instruction — "add error handling," "convert to a function," "add comment blocks" — and Claude applies the change in context, understanding the surrounding code.

Inline suggestions also appear automatically as you type in supported file types. Accept a suggestion with Tab, dismiss with Escape.

Terminal Integration

Claude can observe your VS Code integrated terminal. When a command fails, you can paste the error into the chat panel and Claude sees both the error and the code you were running — providing more accurate diagnosis than if you described the error in isolation.

You can also ask Claude to generate a command and it will offer to run it directly in the terminal, with your approval.

/code-review Skill

Type /code-review in the chat panel to trigger a structured code review against your current git diff. Claude analyses changes for bugs, security issues, style inconsistencies, and potential improvements — and can post findings as inline comments or apply fixes directly.

This is particularly useful before committing a new automation script or pushing a configuration change for peer review.

2.4 When to Use Each Interface

Choosing the right interface is not about which is "best" — each excels at different things. As you grow comfortable with all three, you will naturally reach for the right one without thinking about it. Use this table as a reference until that instinct develops.

They complement rather than replace each other.

2.5 Practical Tips

Desktop App Tips

  • Use the Desktop App for long research or multi-document tasks. If you are reading through several network diagrams, comparing firewall policies, or summarising a stack of vendor documents — attach them all and work through the analysis in a single conversation.
  • Name your conversations meaningfully. Click the conversation title to rename it. "AD group audit June 2026" is much more useful than "New Chat" when you need to find it three weeks later.
  • Create one Project per major area of work. Consider Projects like "Server Infrastructure," "Helpdesk Scripts," and "Network Changes." Each gets its own system prompt describing the relevant context.

CLI Tips

  • Pipe log files directly for fast triage. Do not copy-paste — pipe:
PowerShell — Pipe event log to Claude Example 
# Pipe the last 50 system events to Claude for error identification
Get-EventLog -LogName System -Newest 50 |
    Format-List -Property TimeGenerated, EntryType, Source, Message |
    Out-String |
    claude "Identify any errors or warnings that need attention. List them by severity."

# Check failed login attempts from the last hour
Get-WinEvent -FilterHashtable @{LogName='Security'; Id=4625; StartTime=(Get-Date).AddHours(-1)} |
    Select-Object TimeCreated, Message |
    ConvertTo-Json |
    claude "Summarise these failed login events. Are there any patterns suggesting a brute force attempt?"

# Analyse a text log file
Get-Content "C:\Logs\iis\u_ex260628.log" | claude "Find any 500 errors and summarise the affected URLs"
  • Use the REPL for iterative work. Run claude (no arguments) to open an interactive session when you need to have a back-and-forth — developing a script step by step, for example.
  • Store useful system prompts in a .txt file. If you frequently use a particular system prompt (e.g., your IT environment description), save it and reference it: claude --system (Get-Content ".\system-prompt.txt" | Out-String)
Tip
The CLI is installed as claude, but Anthropic's documentation may refer to it as Claude Code. When you see "run Claude Code" or "use Claude Code" in official materials, that means run the claude command from your terminal. The two names refer to the same installed tool.

VS Code Tips

  • Open the full project folder, not just a single file. Claude's power in VS Code comes from workspace context. Use Ctrl+K, Ctrl+O to open your scripts repository root so Claude can understand how your files relate to each other.
  • Use /code-review before committing. Make it a habit to run the code review skill after finishing any significant script or configuration file. It catches issues that are easy to miss when you are deep in the implementation details.
  • Describe the goal, not just the fix. Instead of "fix line 24," try "this script is supposed to sync user accounts from CSV to AD — what's wrong with the logic on line 24 that would cause it to skip disabled accounts?" More context produces better suggestions.
  • Pair with the terminal integration for live debugging. When a script fails in the VS Code terminal, immediately switch to the Claude panel and paste the error — Claude sees both the error output and the source file and can diagnose across both.

Security Reminders Across All Interfaces

Data Handling — Important

Before pasting any content into Claude — in any interface — consider what that content contains:

  • Do not paste passwords, API keys, or credentials of any kind. Redact them before sharing logs or configs.
  • Student personal data (names, ID numbers, grades) is governed by FIPPA and institutional policy. Do not paste records that identify individual students.
  • Sensitive system details (internal IP ranges, AD structure, firewall rules) should only be shared in conversations protected by your institution's approved access tier. Consult the Technology Services data classification policy.
  • Claude conversations on the Pro/Teams plan are not used to train Anthropic's models by default, but always confirm the current policy with Technology Services before sharing sensitive infrastructure details.
